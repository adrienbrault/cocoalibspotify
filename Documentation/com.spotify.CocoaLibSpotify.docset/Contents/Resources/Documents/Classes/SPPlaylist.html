Overview

This class represents a list of tracks, be it a user’s starred list, inbox, or a traditional “playlist”.

offlineStatus

Returns the offline status of the playlist. Possible values:

@property (readonly) sp_playlist_offline_status offlineStatus

Discussion

SP_PLAYLIST_OFFLINE_STATUS_NO
Playlist is not offline enabled.

SP_PLAYLIST_OFFLINE_STATUS_YES
Playlist is synchronized to local storage.

SP_PLAYLIST_OFFLINE_STATUS_DOWNLOADING
This playlist is currently downloading. Only one playlist can be in this state any given time.

SP_PLAYLIST_OFFLINE_STATUS_WAITING
Playlist is queued for download.

Parameters

The playlist URL to create an SPPlaylist for.

aSession

The SPSession the playlist should exist in.

Return Value

Returns the created SPPlaylist object.

Discussion

This convenience method creates an SPPlaylist object if one doesn’t exist, or returns a cached SPPlaylist if one already exists for the given URL.

Important: If you pass in an invalid playlist URL (i.e., any URL not starting spotify:user:XXXX:playlist:, this method will return nil.

moveTracksAtIndexes:toIndex:error:

Move track(s) to another location in the list.

- (BOOL)moveTracksAtIndexes:(NSIndexSet *)indexes toIndex:(NSUInteger)newLocation error:(NSError **)error

Parameters

indexes

The indexes of the tracks to move.

newLocation

The index the tracks should be moved to.

error

An NSError pointer to be filled if the operation fails.

Return Value

Returns YES if the operation succeeded, otherwise NO.

Discussion

All indexes are given relative to the state of the track order before the move is executed. Therefore, you don’t need to adjust the destination index to take into account tracks that will be moved from above it.

Important: This operation can fail, for example if you give invalid indexes. Please make sure you check the result of this method.
