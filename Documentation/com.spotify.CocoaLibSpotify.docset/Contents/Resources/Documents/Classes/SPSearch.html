Overview

searchWithSearchQuery:inSession:

Creates a new SPSearch with the default page size for the given query.

+ (SPSearch *)searchWithSearchQuery:(NSString *)searchQuery inSession:(SPSession *)aSession

Parameters

searchQuery

The search query to create an SPSearch for.

aSession

The SPSession the track should exist in.

Return Value

Returns the created SPSearch object.

Discussion

This convenience method is simply returns a new, autoreleased SPSearch object. No caching is performed.

Important: The search query will be sent to the Spotify search service immediately. Be sure you want to perform the search before creating the object!

searchWithURL:inSession:

Creates a new SPSearch with the default page size from the given Spotify search URL.

+ (SPSearch *)searchWithURL:(NSURL *)searchURL inSession:(SPSession *)aSession

Parameters

searchURL

The search URL to create an SPSearch for.

aSession

The SPSession the track should exist in.

Return Value

Returns the created SPSearch object.

Discussion

This convenience method is simply returns a new, autoreleased SPSearch object. No caching is performed.

Important: If you pass in an invalid search URL (i.e., any URL not starting spotify:search:, this method will return nil.

Important: The search query will be sent to the Spotify search service immediately. Be sure you want to perform the search before creating the object!

addAlbumPage

Request an extra page of albums from the Spotify search service.

- (BOOL)addAlbumPage

Return Value

Returns YES if a search request was created, NO if there are no more results to fetch or if a search is already in progress.

Discussion

The albums property will be updated when new results are returned.

Important: If you want to request more than just albums, use addPageForArtists:albums:tracks: for better performance.

Parameters

searchAlbum

searchTrack

Return Value

Discussion
